// channels

Durable message queues

@moleculer/channels equivalent — unlike emit/broadcast which are fire-and-forget, channels persist messages until ACK'd.

Producer → Consumer flow

Messages are stored until successfully ACK'd. Consumer groups share load — each message goes to exactly one consumer within a group.

broker.send_to_channel() → Adapter (in-mem) → Consumer Group

Failure handling

NACK triggers exponential backoff retry. After max_retries failures, the message is moved to the Dead Letter Queue.

Handler NACK Retry + backoff → DLQ

Code example

1// Subscribe to durable channel
2broker.subscribe_channel(
3 ChannelDef::new("orders.created", |msg: ChannelMessage| async move {
4 // process order...
5 msg.ack().await // or msg.nack() to retry
6 })
7 .group("order-processor")
8 .max_in_flight(5)
9 .max_retries(3)
10 .dead_letter("orders.DLQ"),
11).await.unwrap();

Get running in 30 seconds

Add moleculer-rs to your Cargo project and spin up a full microservices node.

1. Add to Cargo.toml

[dependencies]
moleculer = "0.1"
tokio = "{ version = \"1\", features = [\"full\"] }"

2. Clone and run the demo

$ git clone https://github.com/oneErrortime/mj
$ cd mj
$ cargo run
​
# Then open the Lab UI:
$ open https://oneerrortime.github.io/mj/lab/

3. Connect the Lab UI to your instance

# In the Lab UI, set endpoint to:
http://localhost:3210

# Live data: topology, metrics, traces, channels, logs
